# Requirements Archive: Dev Journal v1.0

**Archived:** 2026-05-27
**Milestone:** v1.0 MVP
**Defined:** 2026-05-21
**Core Value:** A fast place to write and later find anything you've worked on — entries writable in seconds and searchable in seconds

---

## v1 Requirements — All Delivered ✅

### Entries

- [x] **ENTR-01**: User can create a new journal entry with a title and free-form Markdown body
  *Delivered in Phase 3 (03-02, 03-03) — `createEntry()` Server Action + EditorForm with `@uiw/react-md-editor`*
- [x] **ENTR-02**: User can edit any existing entry
  *Delivered in Phase 3 (03-02, 03-03) — `updateEntry()` Server Action + `/entries/[id]/edit` page*
- [x] **ENTR-03**: User can delete an entry with a confirmation prompt
  *Delivered in Phase 3 (03-04) — `deleteEntry()` Server Action + `DeleteButton` with `shadcn/ui AlertDialog`*
- [x] **ENTR-04**: Entry changes are auto-saved while writing (debounced ~500ms) — no manual Save button required
  *Delivered in Phase 3 (03-03) — `EditorForm` autosave with 2s debounce + race condition guard*

### Tags

- [x] **TAG-01**: User can add one or more project tags to an entry
  *Delivered in Phase 3 (03-02, 03-03) — `setEntryTags()` Server Action + `EditorTagChip` add/remove UI*
- [x] **TAG-02**: Tags are automatically normalized (trimmed, lowercased, deduplicated, empty values removed)
  *Delivered in Phase 3 (03-01, 03-02) — `normalizeTag()` utility with double-normalization guard (client + server)*

### Viewing

- [x] **VIEW-01**: User sees a list of all entries on the home page, newest first, with date and preview snippet
  *Delivered in Phase 2 (02-01, 02-02) — `getEntries()` LEFT JOIN + Map reduce + `EntryCard` component*
- [x] **VIEW-02**: User can open an entry to read it fully rendered as formatted Markdown
  *Delivered in Phase 2 (02-01, 02-02) — `getEntryById()` + `MarkdownBody` with `react-markdown` + `remark-gfm`*
- [x] **VIEW-03**: Fenced code blocks in entries render with syntax highlighting
  *Delivered in Phase 2 (02-01) — `rehype-highlight` + `highlight.js/styles/github-dark.css`*

### Search

- [x] **SRCH-01**: User can search entries by keyword with live results (updates on keypress, debounced)
  *Delivered in Phase 4 (04-01, 04-02, 04-03) — `searchEntries()` FTS5 keyword branch + `SearchInput` 300ms debounce*
- [x] **SRCH-02**: User can filter entries by tag to see all entries with that tag
  *Delivered in Phase 4 (04-01, 04-02, 04-03) — `searchEntries()` tag-only branch + `TagChip` navigating button + `ActiveFilterChip`*
- [x] **SRCH-03**: User can combine keyword search and tag filter simultaneously
  *Delivered in Phase 4 (04-01, 04-03) — `searchEntries()` combined branch (FTS5 MATCH + tag JOIN) + URL state*

## Out of Scope (Confirmed)

| Feature | Reason | Still Valid? |
|---------|--------|-------------|
| Authentication / login | Single-user local app, no auth needed | ✓ Yes |
| Multi-user support | Personal tool only | ✓ Yes |
| Cloud deployment | localhost only for v1 | ✓ Yes |
| Rich text / WYSIWYG editor | Markdown is preferred | ✓ Yes |
| File / image attachments | Adds storage complexity | ✓ Yes |
| Nested tags or tag hierarchy | Flat tags + FTS search eliminates need | ✓ Yes |
| Revision history | Adds complexity disproportionate to solo use | ✓ Yes |
| Dark mode | Can add later; not core to v1 | ✓ Yes |
