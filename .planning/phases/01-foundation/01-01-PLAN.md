    Define three Drizzle SQLite tables using `sqliteTable` from `drizzle-orm/sqlite-core`:
    - `entries`: id (integer, PK, autoIncrement), title (text, notNull, default ''), body (text, notNull, default ''), createdAt (integer, mode: 'timestamp', notNull, $defaultFn new Date()), updatedAt (integer, mode: 'timestamp', notNull, $defaultFn new Date()).     - `tags`: id (integer, PK, autoIncrement), name (text, notNull, unique).
    - `entryTags` (table name 'entry_tags'): entryId (integer, notNull, references entries.id with onDelete: 'cascade'), tagId (integer, notNull, references tags.id with onDelete: 'cascade').     Do NOT define the FTS5 virtual table in schema.ts — Drizzle has no type for virtual tables. FTS5 is created exclusively via raw SQL in initSchema() (see File 2).

    **File 2: src/lib/db/index.ts** (per D-04, D-06)
    Implement the globalThis singleton pattern from Pattern 1 in RESEARCH.md exactly:
    - Declare `global { var __db: Database.Database | undefined }` to satisfy TypeScript's strict globalThis typing.
    - `initSchema(sqlite)`: calls `sqlite.exec(...)` with the full raw SQL string. The SQL must include: `CREATE TABLE IF NOT EXISTS entries`, `CREATE TABLE IF NOT EXISTS tags`, `CREATE TABLE IF NOT EXISTS entry_tags`, `CREATE VIRTUAL TABLE IF NOT EXISTS entries_fts USING fts5(title, body, content=entries, content_rowid=id)`, and three triggers: `entries_fts_insert` (AFTER INSERT), `entries_fts_update` (AFTER UPDATE — delete old row, insert new), `entries_fts_delete` (AFTER DELETE). All use `CREATE TRIGGER IF NOT EXISTS`. See the FTS5 Content Table + Triggers code example in RESEARCH.md for exact SQL syntax.
    - `getDb()`: checks `globalThis.__db`. If undefined: constructs `dbPath` via `path.join(process.cwd(), '.data', 'journal.db')`, calls `fs.mkdirSync(path.dirname(dbPath), { recursive: true })` to ensure `.data/` exists, creates `new Database(dbPath)`, sets WAL journal_mode and foreign_keys pragmas, calls `initSchema(sqlite)`, assigns to `globalThis.__db`.
    - Export: `export const db = drizzle(getDb(), { schema })` where schema is imported as `import * as schema from './schema'`.

    **File 3: drizzle.config.ts** (per D-03, at project root)
    Use `defineConfig` from `drizzle-kit`. Set: dialect 'sqlite', schema './src/lib/db/schema.ts', dbCredentials.url '.data/journal.db'. This config is used exclusively by drizzle-kit CLI commands — it is not imported by application code.

    Create the `.data/` directory at the project root: `mkdir -p .data`. The directory must exist for drizzle-kit push to work even though db/index.ts also creates it at runtime.
