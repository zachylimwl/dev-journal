## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03 | Tampering | /entries/[id] param — path traversal | mitigate | Number(id) coerces non-numeric strings to NaN; Drizzle .where(eq(entries.id, NaN)) returns empty rows; notFound() fires. No DB error, no raw file access. Covered by acceptance criteria: entries/abc returns 404. |
| T-02-04 | Tampering | XSS via stored Markdown in body field | mitigate | react-markdown renders Markdown string to React elements via the reconciler — no dangerouslySetInnerHTML. HTML tags in Markdown are sanitized by default (react-markdown strips unknown elements). No bypass possible through MarkdownBody as implemented. |
| T-02-05 | Information Disclosure | Error boundary — unhandled DB failure | accept | Phase 2 does not add custom error.tsx. Unhandled exceptions bubble to Next.js default error boundary (shows generic error page, not stack trace in production). Acceptable for localhost personal tool. |
| T-02-SC | Tampering | npm packages already installed in Plan 01 | accept | Packages installed in Plan 01 (react-markdown, remark-gfm, rehype-highlight, @tailwindcss/typography). All from official orgs. Covered by T-02-SC in Plan 01 threat model. |
