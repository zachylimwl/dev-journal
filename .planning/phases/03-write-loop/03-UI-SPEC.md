## Spacing Scale

Declared values (all multiples of 4px):

| Token | Value | Usage |
|-------|-------|-------|

| sm | 8px | Gap between chips in tag zone (`gap-2`) |
| md | 16px | Horizontal page padding (`px-4`), input horizontal padding |
| lg | 24px | Vertical section separation within editor page |
| xl | 32px | Vertical page padding (`py-8`) |

| 3xl | 64px | Not used in Phase 3 |

Exceptions:
- Touch targets for the × dismiss button on editor TagChips: minimum 44px hit area via negative margin or padding expansion, even though the visual chip is smaller.
- `py-5` (20px) on EntryCard — carry forward from Phase 2, not changed in Phase 3.

## Typography

Carry forward from Phase 2. Phase 3 uses 4 type roles.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 24px (`text-2xl`) | 600 (`font-semibold`) | 1.2 | App header "Dev Journal" wordmark; editor title input |
| Body | 16px (`text-base`) | 400 (`font-normal`) | 1.5 | Editor title input text, AlertDialog body text |
| Small | 14px (`text-sm`) | 400 (`font-normal`) | 1.5 | Date labels, back link, snippet text, autosave status label |
| Label | 12px (`text-xs`) | 400 (`font-normal`) | 1.4 | TagChip text, date metadata on cards |

**Out of scope for Phase 3:** Heading (30px) — used only on the Phase 2 read-only detail page `<h1>` title; Subheading (18px) — entry card titles (`<h2>`) introduced in Phase 2. Neither role is introduced in Phase 3 screens.

**Font weights declared:** 400 (`font-normal`) for body/small/label/chip text; 600 (`font-semibold`) for display. No intermediate weight.

**Editor title input:** `text-2xl font-semibold` to visually match the detail page heading — the field IS the title, not a form label above a text box.

**Autosave status:** `text-xs text-zinc-400` (idle-fade), `text-xs text-zinc-500` (active states). No bold weight — must read as ambient, not status bar.

## Color

The project palette is zinc-based. All values carried from Phase 2 source code.

| Role | Tailwind Token | CSS Approximate | Usage |
|------|---------------|-----------------|-------|
| Dominant (60%) | `bg-white` / `--background: #ffffff` | #ffffff | Page background, editor surface |
| Secondary (30%) | `bg-zinc-100` | #f4f4f5 | TagChip background, input focus ring base, header border fill |
| Accent (10%) | `text-zinc-900` / `border-zinc-200` | #18181b / #e4e4e7 | Primary interactive borders, header text |
| Destructive | `bg-red-600 hover:bg-red-700` | #dc2626 / #b91c1c | AlertDialog "Delete" confirm button only |
| Muted text | `text-zinc-400` / `text-zinc-500` / `text-zinc-600` | — | Date labels, snippets, autosave label |

Accent reserved for:
- "New Entry" button in AppHeader (zinc-900 background, white text)
- "Edit" button on detail page (zinc-100 background, zinc-800 text — secondary action styling)
- Active / focused state on tag input field border
- AlertDialog "Cancel" button (zinc-100 / zinc-800)

Destructive reserved for:
- AlertDialog "Delete entry" confirm button only
- "Delete" trigger button on detail and edit pages: `text-red-600 hover:text-red-700` (text-only, no fill, until AlertDialog opens)

**Do not use red on any non-destructive element.**

## Component Specifications

### "New Entry" button — AppHeader

- Position: right slot of header (`justify-between` already in place)
- Appearance: `px-4 py-2 rounded-md text-sm font-medium bg-zinc-900 text-white hover:bg-zinc-700`
- Label: "New Entry"
- Behavior: `<Link href="/new">` — no client interaction needed

### Editor page layout — `/new` and `/entries/[id]/edit`

Both pages share an `EditorForm` client component. Layout within `<main className="max-w-5xl mx-auto px-4 py-8">`:

```
[ ← Back link                    ] [ Saving... / Saved just now / (empty) ]
[ Title input — full width                                                 ]
[ Tag zone: [chip ×] [chip ×] [__tag input________]                       ]
[ Markdown editor — full width, ~500px min height                         ]
[ (no Save button)                                                         ]
```

**Back link:** `text-sm text-zinc-500 hover:text-zinc-800` — "← All entries" on `/new`; "← Back to entry" on `/entries/[id]/edit` (links to `/entries/[id]`)

**Header row:** `flex items-center justify-between mb-4`
- Left: back link
- Right: autosave status label (see below)

**Title input:**
- `w-full text-2xl font-semibold text-zinc-900 placeholder:text-zinc-300 border-0 border-b border-zinc-200 focus:border-zinc-400 focus:outline-none pb-2 mb-4 bg-transparent`
- Placeholder text: "Entry title"
- No visible label element — the input IS the title

**Tag zone:** `flex flex-wrap items-center gap-2 mb-4`
- Existing tags: `TagChip` component + inline `×` button
- `×` button: `ml-1 text-zinc-400 hover:text-zinc-700 text-xs leading-none` — "×" character (not an SVG icon)
- Tag input field: `text-sm text-zinc-700 placeholder:text-zinc-400 border-0 focus:outline-none bg-transparent min-w-[120px]` — borderless, inline with chips
- Placeholder: "Add tag…"
- Behavior: Enter or comma creates chip; input clears; tag is normalized immediately on creation

**Markdown editor:** `@uiw/react-md-editor` via `dynamic()` with `ssr: false`. Full width. Minimum height 500px. No additional wrapper border — the editor provides its own chrome.

### Autosave status label

- Position: top-right of editor header row, right-aligned in the flex header
- States:
  - Saving: `text-xs text-zinc-500` — text "Saving..."
  - Saved: `text-xs text-zinc-500` — text "Saved just now" — transitions to fade-out after 2500ms
  - Idle: `text-xs text-zinc-300 opacity-0` — empty string or hidden (faded out) — no layout shift; keep element in DOM with `opacity-0 transition-opacity`
- Transition: `transition-opacity duration-500` — fade from visible to invisible over 500ms starting 2500ms after "Saved just now" appears
- Never uses bold, never uses color other than zinc-500/zinc-300
- Never shows an icon — text only

### TagChip in editor context (EditorTagChip)

Extend `TagChip` with an `onRemove` prop for editor-only use. Visual chip is identical to Phase 2 `TagChip`:
- `inline-flex items-center px-2 py-0.5 rounded-full text-xs font-normal bg-zinc-100 text-zinc-600`
- Append `×` button inside the chip: `ml-1.5 text-zinc-400 hover:text-zinc-700 cursor-pointer leading-none`
- Hit area: wrap `×` in a `<button>` with `p-1 -m-1` to expand touch target to ~28px minimum

### Edit and Delete buttons — detail page `/entries/[id]`

Add below the tag row, above the `MarkdownBody`:

```
[ Edit ]  [ Delete ]
```

- Layout: `flex gap-3 mt-4`
- "Edit" button: `<Link>` styled as `px-3 py-1.5 rounded-md text-sm font-medium bg-zinc-100 text-zinc-800 hover:bg-zinc-200`
- "Delete" button: `<button>` styled as `px-3 py-1.5 rounded-md text-sm font-medium text-red-600 hover:text-red-700 hover:bg-red-50` — opens AlertDialog on click
- Same button pair appears on `/entries/[id]/edit` below the editor, with identical styling

### AlertDialog — delete confirmation

Uses shadcn/ui `AlertDialog` component (Radix UI primitive). Copy exact wording:

- **Title:** "Delete this entry?"
- **Description:** "This action cannot be undone. The entry will be permanently removed."
- **Cancel button:** "Cancel" — `variant="outline"` (zinc-100 background, zinc-800 text)
- **Confirm button:** "Delete entry" — `variant="destructive"` (red-600 background, white text)
- Destructive button is positioned right of Cancel
- On confirm: calls `deleteEntry(id)` Server Action, then redirects to `/`
- On cancel: dialog closes, no action taken

## Interaction Contract

| Interaction | Trigger | Behavior |
|-------------|---------|----------|
| Create entry | Click "New Entry" in header | Navigate to `/new` |
| First autosave (new entry) | 500ms after first keystroke in title or body | `createEntry()` Server Action → redirect to `/entries/[id]/edit` |
| Autosave (existing entry) | 500ms debounce after any keystroke in title, body, or tag change | `updateEntry()` Server Action → status transitions Saving → Saved → fade |
| Add tag | Press Enter or comma in tag input | Normalize tag (trim, lowercase) → create chip → clear input → trigger autosave |
| Remove tag | Click × on chip | Remove chip from local state → trigger autosave |
| Navigate to edit | Click "Edit" on detail page | Navigate to `/entries/[id]/edit` |
| Trigger delete | Click "Delete" on detail or edit page | Open AlertDialog |
| Confirm delete | Click "Delete entry" in AlertDialog | `deleteEntry(id)` → redirect to `/` |
| Cancel delete | Click "Cancel" or press Escape | Close AlertDialog, no action |
| Empty body navigation | User visits `/new` and navigates away before typing | No DB record created — safe abandon |
