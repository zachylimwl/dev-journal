<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input → EditorForm state | Title, body, tag input from keyboard events |
| EditorForm → Server Action | title, body, tagNames passed to createEntry / updateEntry / setEntryTags |
| Server Action → SQLite | Content written via Drizzle parameterized queries |
| /entries/[id]/edit → DB | Entry loaded at render time via getEntryById; id from URL params |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-06 | Spoofing/Tampering | Markdown body rendered back via MDEditor — XSS via imported .md file | mitigate | MDEditor edit mode only shows editor chrome; read view uses react-markdown (no dangerouslySetInnerHTML); XSS cannot execute in editor pane |
| T-03-07 | Tampering | entryId in URL params for /entries/[id]/edit — ID manipulation | accept | Single-user local app; no authorization boundary; user has access to all their own entries |
| T-03-08 | Denial (data quality) | Fast typing triggers multiple createEntry calls before first returns | mitigate | isSavingRef guard in triggerAutosave prevents concurrent saves; entryIdRef sentinel prevents duplicate DB records |
| T-03-09 | Tampering | Tag normalization bypass — client sends un-normalized tags | mitigate | Server-side normalization in setEntryTags (Plan 02) is the authoritative guard; client-side normalization in addTag() is UX convenience only |
| T-03-SC | Tampering | npm/CDN packages pulled during dynamic import | accept | @uiw/react-md-editor already installed in node_modules at Wave 0; no runtime CDN fetches |
</threat_model>
