<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → Server Action | createEntry/updateEntry/deleteEntry/setEntryTags receive user-supplied strings |
| Server Action → SQLite | Tag names and entry content written to DB via Drizzle parameterized queries |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-02 | Tampering | Tag names in setEntryTags — malicious SQL via tag string | mitigate | Drizzle parameterized queries: tag name passed as bound parameter to db.insert(tags).values({ name }); never string-interpolated into SQL |
| T-03-03 | Tampering | Entry title/body in createEntry/updateEntry — SQL injection | mitigate | Drizzle .values({ title, body }) uses parameterized binding; no raw SQL construction |
| T-03-04 | Denial (data quality) | Orphan empty entries on /new page load | mitigate | createEntry called only from Client Component debounce; content guard (!t.trim() && !b.trim()) in EditorForm (Plan 03) prevents calling createEntry for empty content; action itself does not enforce this — the guard lives in the caller |
| T-03-05 | Information Disclosure | Server Actions exposed as POST endpoints | accept | Single-user localhost app; no auth required per project constraints; no PII exposure risk |
</threat_model>
