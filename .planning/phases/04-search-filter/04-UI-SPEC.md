## Component Inventory

New and modified components this phase:

### 1. SearchInput (`src/components/search-input.tsx`)

- `'use client'` directive
- Renders: `<input type="search">` wrapped in a relative `<div>` with a search icon (Lucide `Search`, 16px) positioned left inside the input
- Width: full width of content column (`w-full`)
- Height: 40px (h-10, py-2 px-3 with pl-9 to clear the icon)
- Border: `border border-input` (--border: oklch(0.922 0 0), matches existing form inputs)
- Border radius: `rounded-md` (--radius-md)
- Focus state: `focus:outline-none focus:ring-2 focus:ring-ring focus:border-transparent` — ring uses --ring variable
- Placeholder text: "Search entries..." (14px, text-muted-foreground)
- Behavior: debounced 300ms `router.push` updating `?q=` param; preserves existing `?tag=` param
- defaultValue prop: pre-filled from `searchParams.q` on server render

### 2. ResultCountLabel (inline in `src/app/page.tsx`, not a separate component)

- Renders only when `q` or `tag` searchParam is set
- Copy: `"{N} entries"` — singular: `"1 entry"`, plural: `"N entries"`
- Typography: text-sm (14px), text-muted-foreground (--muted-foreground: oklch(0.556 0 0))
- Placement: below SearchInput, above ActiveFilterChip (if present), above entry list
- Margin: `mt-3` (12px, acceptable exception between sm=8 and md=16 to match existing gap-1.5 patterns)

### 3. ActiveFilterChip (`src/components/active-filter-chip.tsx` or inline in page.tsx)

- Renders only when `?tag` param is set
- Layout: `inline-flex items-center gap-1.5 px-3 py-1 rounded-full`
- Background: `bg-zinc-100` (matches TagChip — visual family consistency)
- Text: "Active filter: {tagName}" — text-xs (12px), font-medium (500), text-zinc-600
- × button: Lucide `X` icon at 12px, text-zinc-400, hover:text-zinc-700; clicking clears `?tag=` param from URL (preserves `?q=` if set)
- × button touch target: min 32px via `p-1` wrapper around the icon
- Placement: below ResultCountLabel (if shown), above entry list; `mt-2` below count label

### 4. TagChip (`src/components/tag-chip.tsx`) — upgraded

- Adds `'use client'` directive
- Wraps content in `<button>` instead of `<span>`
- onClick: `router.push('/?tag=' + encodeURIComponent(name))`
- Cursor: `cursor-pointer`
- Visual: unchanged — `inline-flex items-center px-2 py-0.5 rounded-full text-xs font-medium bg-zinc-100 text-zinc-600`
- Hover: `hover:bg-zinc-200 hover:text-zinc-900` — subtle darkening, no outline
- Active state when current page tag matches: no special styling in Phase 4 (deferred)

## Interaction Contract

| Interaction | Trigger | Result |
|-------------|---------|--------|
| Type in search box | keypress (debounced 300ms) | `router.push('/?q={value}&tag={existing}')` — preserves tag |
| Clear search box | backspace to empty or click × in native search input | `router.push('/?tag={existing}')` — removes q param |
| Click TagChip | click | `router.push('/?tag={name}')` — replaces tag param, preserves q |
| Click × on ActiveFilterChip | click | `router.push('/?q={existing}')` — removes tag param, preserves q |
| Navigate back | browser back | URL restores previous search state (bookmarkable by design) |
| Empty FTS5 guard | q param is empty string or whitespace | `searchEntries` branches to `getEntries()` — no MATCH clause, no crash |
| Search with only tag | q absent, tag present | `searchEntries(null, tag)` — SQL filters on tag join, no FTS5 call |
| Combined search+filter | both q and tag present | `searchEntries(q, tag)` — FTS5 MATCH + tag JOIN, BM25 ranked |

Debounce delay: 300ms (Claude's discretion, D per CONTEXT.md). Shorter than autosave (500ms). Provides live-search feel without excessive navigation events.

## Accessibility Contract

- `<input type="search">` has `aria-label="Search entries"` (no visible label per design — aria-label required)
- × button on ActiveFilterChip has `aria-label="Clear tag filter"`
- TagChip `<button>` has implicit accessible name from text content; no additional aria needed
- Result count `<p>` uses `aria-live="polite"` so screen readers announce result changes after navigation
- Focus order: SearchInput → ActiveFilterChip × → entry list
