## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input → URL params | SearchInput pushes user-typed text to URL; TagChip pushes tag names from server-rendered data |
| URL params → React render | tag and q values from URL appear in ActiveFilterChip text; React escapes by default |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-05 | Tampering | SearchInput — q value rendered in URL | accept | q is pushed to URL, never rendered as HTML directly in this component; no dangerouslySetInnerHTML; XSS risk is nil |
| T-04-06 | Tampering | ActiveFilterChip — tag name rendered as text | accept | React renders {tag} as escaped text node; no XSS vector; tag names originate from server-rendered DB data |
| T-04-07 | Tampering | TagChip — encodeURIComponent on tag name | mitigate | encodeURIComponent() prevents tag names with & or = from corrupting the URL param string |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | No new packages in this phase — package legitimacy gate not applicable |
</threat_model>
