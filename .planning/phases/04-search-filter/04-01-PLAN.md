## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| URL params → Server Action | q and tag arrive from searchParams (user-controlled URL) into searchEntries() |
| searchEntries → SQLite FTS5 | Sanitized q passed to MATCH via parameterized prepare().all(?); tag passed via ? binding |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | FTS5 MATCH in searchEntries | mitigate | Parameterized query (prepare().all(?)) prevents SQL injection. Input sanitization strips FTS5 syntax operators before MATCH to prevent syntax errors. Double defense: params + sanitize. |
| T-04-02 | Tampering | tag param in SQL IN() / Drizzle WHERE | mitigate | Tag value passed via Drizzle eq() or prepare().all(?) — parameterized; never string-interpolated into SQL. |
| T-04-03 | Denial of Service | FTS5 unbounded results | mitigate | LIMIT 200 on all FTS5 raw SQL queries; Drizzle tag-only path uses .limit(200). |
| T-04-04 | Tampering | npm/pip/cargo installs | accept | No new packages in this phase — package legitimacy gate not applicable. |
</threat_model>
